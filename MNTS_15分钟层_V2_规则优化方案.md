# MNTS 15分钟层 V2 规则优化方案

## 目标

这份文档用于定义 `15分钟层 V2` 的优化方向。

V1 已经完成了一个最小闭环验证，证明：

- `cohesion / drift / fracture` 三态不是空概念
- 三态后面的未来波动率存在差异
- `15分钟层` 可以先作为状态机原型成立

但 V1 仍然存在明显不足：

- 对极端高波动事件的区分度不够强
- `drift` 更像兜底类，定义不够积极
- 主要依赖静态阈值，状态边界较硬

因此 V2 的核心目标不是推翻 V1，而是：

- 在保留三态框架的前提下，让 `cohesion / drift / fracture` 更像真正的市场状态
- 让 `fracture` 对不稳定环境更敏感
- 让 `cohesion` 对稳定环境更清晰
- 让整个 `15分钟层` 更接近一个可复用的战术状态机

---

## 一、V1 已经证明了什么

V1 的意义不是“已经能交易”，而是已经证明了：

- 15分钟 Token 分布动力学可以构成一个状态感知层
- `cohesion` 后面未来平均波动率更低
- `fracture` 后面未来平均波动率更高
- `drift` 处于中间地带

这意味着：

- `15分钟层` 的研究方向成立
- 但当前版本还只是第一代原型

---

## 二、V2 需要解决的问题

V2 重点解决 4 个问题：

### 1. fracture 不够锐利

V1 能区分平均未来波动率，但对真正的高波动事件区分度还不够强。

V2 需要让 `fracture` 更接近：

- 状态撕裂
- 结构不稳
- 风险上升

### 2. drift 定义过于被动

V1 中 `drift` 更多是：

- 没有进 `cohesion`
- 也没有进 `fracture`

V2 需要把 `drift` 定义成真正的中间态：

- 市场在变化
- 但变化不是崩裂
- 是慢性迁移而不是剧烈撕裂

### 3. 阈值过硬

V1 主要是：

- 低熵
- 高主导占比
- 高漂移

这些条件的静态组合。

V2 需要从“硬条件”升级到：

- `状态分数`
- 或 `轻量评分器`

### 4. 连续表示还没被利用

V1 只主要用到了：

- 分布漂移
- 熵
- 主导占比

但 `Kronos tokenizer` 的连续表示本身也蕴含异常信息。

V2 需要把“连续表示异常度”纳入。

---

## 三、V2 的总体思路

V2 不直接上复杂模型，不做完整拓扑，也不做一堆高级数学。

V2 的路线是：

- 保留三态框架
- 保留两年数据发现期/验证期
- 保留样本外检验
- 在此基础上增加少量更有信息量的特征
- 然后把静态规则升级成一个轻量风险评分器

也就是说：

- V1 = 手工三态规则
- V2 = 三态评分器

---

## 四、V2 新增的核心特征

V2 不追求特征很多，只加最值得加的几个。

## 4.1 熵变化率

不仅看当前熵的绝对水平，还看：

- 熵最近是上升还是下降

直觉上：

- 熵快速上升，更像撕裂
- 熵持续下降，更像凝聚

---

## 4.2 状态切换速度

观察最近窗口里：

- token 切换是否更频繁

这描述的是：

- 市场是不是开始“坐不住”

切换速度快，通常更容易对应：

- fracture

---

## 4.3 连续表示异常度

使用 `Kronos tokenizer` 的连续表示，构造：

- 当前窗口 embedding 相对于发现期“正常中心”的偏离程度

这个指标的意义是：

- 不只是看 token 标签变没变
- 还看状态空间本身是不是开始异常偏移

---

## 五、V2 的评分器思路

V2 不一开始上重模型，而是做一个非常轻的评分器。

### 输入

- 分布漂移
- 熵
- 熵变化率
- 主导占比
- 状态切换速度
- embedding 异常度

### 目标

先只定义一个目标：

- `未来16根是否进入高波动前10%`

### 输出

得到一个：

- `instability_score`

然后再按分位数切成三态：

- 低分：`cohesion`
- 中分：`drift`
- 高分：`fracture`

这样做的优点是：

- 保留三态解释性
- 又比 V1 更平滑

---

## 六、为什么先只做“高波动目标”

因为当前阶段最重要的是：

- 让 `15分钟层` 先把风险层面的状态识别做扎实

所以 V2 先不同时优化：

- 上涨机会
- 下跌机会
- 假突破
- 趋势质量

而先盯住一个最清晰的目标：

- `未来波动是否上升`

这个目标最适合拿来验证：

- `fracture` 是否真的更像危险状态

---

## 七、V2 的三态定义

在 V2 中，三态不再直接由硬阈值决定，而由评分器决定。

### cohesion

应当表现为：

- 漂移低
- 熵低或持续下降
- 主导占比高
- 切换速度低
- embedding 异常度低

### drift

应当表现为：

- 中等漂移
- 中等熵
- 状态在迁移，但没有失稳

### fracture

应当表现为：

- 漂移高
- 熵偏高或快速上升
- 主导占比低
- 切换速度高
- embedding 异常度高

---

## 八、V2 的验证标准

V2 是否成功，不看概念，而看结果。

至少要满足下面几条中的大部分：

### 1. fracture 的未来平均波动率明显高于 drift

不只是略高，而要更清晰。

### 2. cohesion 的未来平均波动率明显低于 drift

真正体现“凝聚态”的稳定性。

### 3. 高波动事件率的分离度比 V1 更强

这是 V2 最重要的硬标准。

### 4. 三态样本占比不过度塌缩

不能所有样本都被判成同一类。

### 5. 解释仍然成立

V2 不能因为更复杂，就失去解释性。

---

## 九、V2 暂时不做什么

为了防止复杂度失控，V2 暂时不做：

- 完整费舍信息矩阵
- 拓扑联动
- 1小时层联动
- 直接交易信号
- 多目标优化

V2 仍然只做一件事：

- `把15分钟层状态机做得更准`

---

## 十、V2 的最小实现方案

实际落地时，V2 建议按照下面步骤实现：

1. 使用两年 `BTCUSDT 15m` 数据
2. 第一年轻量拟合特征标准与评分权重
3. 第二年冻结后做样本外验证
4. 输出：
- `cohesion / drift / fracture` 三态
- 三态未来波动率
- 三态高波动事件率
- 时间轴图
- V1 / V2 对比结果

---

## 十一、一句话总结

如果把这份文档压缩成一句话，就是：

- `15分钟层 V2 的任务不是发明更复杂的概念，而是在保留三态结构的基础上，利用少量更有信息量的特征和一个轻量评分器，让 cohesion / drift / fracture 变得更锐利、更稳定、更接近真实市场状态。`
