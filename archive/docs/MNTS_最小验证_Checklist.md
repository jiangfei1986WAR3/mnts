# MNTS 最小验证 Checklist

## 目标

本轮验证只回答两个问题：

- `tokenizer 输出有没有统计规律`
- `这条路线是否值得继续研究`

本轮不回答以下问题：

- 系统能否盈利
- 完整拓扑球体是否成立
- MNTS 是否可直接进入实盘

---

## 一、数据准备

- 标的：`BTCUSDT`
- 周期：`15m`
- 长度：至少 `1 年`
- 最低字段：`timestamp, open, high, low, close`
- 推荐字段：再加 `volume`
- 建议切分：前 `8-10` 个月观察，后 `2-4` 个月复查稳定性

通过条件：

- 时间连续
- 缺失较少
- 字段格式统一

失败信号：

- 时间戳混乱
- 数据缺口过多
- OHLCV 字段不一致

---

## 二、Tokenizer 输出

优先方案：

- 直接使用预训练 `Kronos tokenizer`

备选方案：

- 先使用伪 tokenizer 做快速对照实验

至少保留两类输出：

- `Token ID`
- `连续表示`

通过条件：

- 能稳定对整段样本输出状态表示

失败信号：

- 输出异常
- 全部塌缩到极少数 token

---

## 三、Token 使用分布

要看什么：

- 每个 token 的出现次数和占比

推荐图形：

- 柱状图

通过条件：

- 有一批主干状态
- 但不是只剩极少数 token

失败信号：

- 严重塌缩
- 或完全均匀得像随机切片

含义：

- 通过说明“状态空间存在”
- 失败说明离散化层可能无效

---

## 四、Token 后续收益分布

要看什么：

- token 出现后未来 `4 / 8 / 16` 根 K 线的收益分布

推荐图形：

- 箱线图
- 小提琴图

通过条件：

- 至少一部分 token 的未来收益分布明显不同

失败信号：

- 各 token 后续收益分布高度重叠

含义：

- 通过说明 token 具有行为区分度

---

## 五、Token 后续波动率分布

要看什么：

- token 出现后未来 `4 / 8 / 16` 根 K 线的实现波动率

推荐图形：

- 箱线图

通过条件：

- 部分 token 明显对应高波动或低波动

失败信号：

- 各 token 后续波动结构无差异

含义：

- 即使方向性不强，只要能区分风险状态，也值得继续

---

## 六、Token 转移矩阵

要看什么：

- 当前 token 到下一 token 的转移概率

推荐图形：

- 热力图

通过条件：

- 存在桥梁态、自循环态、局部状态链

失败信号：

- 转移矩阵近似均匀随机

含义：

- 通过说明“状态语法”存在

---

## 七、滚动分布变化

要看什么：

- 相邻滚动窗口 token 分布之间的变化

建议先用：

- `JS 散度`
- `L1 距离`

推荐图形：

- 时间序列曲线

通过条件：

- 趋势切换、急跌急拉、波动爆发附近有明显响应

失败信号：

- 曲线平得没有信息
- 或乱跳且与市场状态无关

含义：

- 通过说明信息几何方向值得继续

---

## 八、连续表示结构

要看什么：

- 连续表示的降维结构

建议方法：

- `PCA`
- `UMAP`（可选）

推荐图形：

- 2D 散点图

通过条件：

- 不同阶段有初步簇结构、轨迹结构或区域迁移

失败信号：

- 完全糊成一团，没有结构感

含义：

- 通过说明拓扑输入层有继续研究价值

---

## 九、最小通过标准

满足下面任意 `3` 条，就值得继续：

- token 使用分布没有严重塌缩
- token 后续收益分布存在差异
- token 后续波动率分布存在差异
- 转移矩阵有结构
- 滚动分布变化能响应市场切换
- 连续表示有初步簇结构

---

## 十、止损标准

如果多数现象同时出现，建议暂时止损或降级路线：

- token 严重塌缩
- 收益和波动率分布无明显差异
- 转移矩阵像随机噪声
- 分布变化对市场状态不敏感
- 连续表示没有结构

---

## 十一、通过后的下一步

如果通过，不应立刻进入完整系统开发。

更合理的下一步是：

- 把 token 分布变化升级为更严谨的信息几何指标
- 把连续表示用于小规模拓扑点云实验
- 开始验证“结构异常是否真的对应风险上升”

---

## 一句话总结

- 这份 checklist 验证的不是“MNTS 能不能赚钱”，而是“市场是否真的能被 tokenizer 压缩成一套有规律的状态语言”。
